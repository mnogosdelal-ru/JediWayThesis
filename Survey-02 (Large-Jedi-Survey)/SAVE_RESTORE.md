# Сохранение и восстановление данных

## 📋 Описание

Приложение автоматически сохраняет ответы респондента и восстанавливает их при возврате на страницу.

## ✨ Возможности

### 1. Автосохранение при нажатии "Назад"

Когда респондент нажимает кнопку "Назад":
- ✅ Все заполненные данные сохраняются в localStorage браузера
- ✅ При возврате на страницу данные восстанавливаются
- ✅ Показываются индикаторы сохранения и восстановления

### 2. Автосохранение каждые 30 секунд

- ✅ Каждые 30 секунд данные автоматически сохраняются
- ✅ Показывается зелёный индикатор "✓ Сохранено"
- ✅ Работает только на страницах 0-9

### 3. Сохранение при закрытии вкладки

- ✅ При закрытии вкладки/браузера данные сохраняются
- ✅ При возврате можно продолжить с того же места

### 4. Восстановление данных

При загрузке страницы:
- ✅ Проверяется localStorage на наличие сохранённых данных
- ✅ Если данные есть - они восстанавливаются
- ✅ Показывается жёлтое уведомление "ℹ️ Ваши предыдущие ответы восстановлены"
- ✅ Уведомление исчезает через 5 секунд

## 🔧 Как это работает

### localStorage

Данные сохраняются в localStorage браузера:

```javascript
// Ключ: survey_page_0, survey_page_1, и т.д.
localStorage.setItem('survey_page_1', JSON.stringify(formData));
```

### Структура данных

```json
{
  "respondent_id": "abc123",
  "page": "1",
  "next_page": "2",
  "age": "35",
  "gender": "male",
  "children_count": "2",
  ...
}
```

### Очистка данных

Данные очищаются только после успешного сохранения на сервер:

```javascript
// В save.php, после успешного сохранения
localStorage.removeItem(`survey_page_${CURRENT_PAGE}`);
```

## 📊 Индикаторы

### Зелёный индикатор (справа сверху)

```
┌─────────────────┐
│ ✓ Сохранено     │  ← Показывается на 2 секунды
└─────────────────┘
```

**Когда показывается:**
- При автосохранении (каждые 30 сек)
- При нажатии на "Назад"
- При изменении любого поля

### Жёлтое уведомление (под прогрессом)

```
┌────────────────────────────────────────────────────┐
│ ℹ️ Ваши предыдущие ответы восстановлены. Вы можете │
│ продолжить с того места, где остановились.         │
└────────────────────────────────────────────────────┘
```

**Когда показывается:**
- При загрузке страницы с сохранёнными данными
- Исчезает через 5 секунд

## 🎯 Сценарии использования

### Сценарий 1: Случайный переход назад

1. Респондент заполняет страницу 3
2. Случайно нажимает "Назад"
3. Данные сохраняются
4. Респондент возвращается на страницу 3
5. Данные восстанавливаются ✅

### Сценарий 2: Закрытие вкладки

1. Респондент заполняет страницу 5
2. Закрывает вкладку (или браузер)
3. Открывает ссылку снова
4. Данные на странице 5 восстанавливаются ✅

### Сценарий 3: Долгое заполнение

1. Респондент медленно заполняет страницу 8 (Практики)
2. Каждые 30 секунд данные автосохраняются
3. Если браузер "упадёт", данные не потеряются ✅

## 🔒 Безопасность

### Что сохраняется в localStorage:

- ✅ Все ответы на вопросы
- ✅ ID респондента
- ✅ Номер страницы

### Что НЕ сохраняется:

- ❌ Персональные данные (кроме ответов)
- ❌ Пароли
- ❌ Платёжная информация

### Очистка данных:

Данные в localStorage очищаются:
- После успешного сохранения на сервер
- При завершении опросника (страница 10)
- Вручную пользователем (очистка кэша браузера)

## 🐛 Отладка

### Проверить сохранённые данные

Откройте консоль браузера (F12) и выполните:

```javascript
// Проверить данные для страницы 1
console.log(localStorage.getItem('survey_page_1'));

// Показать все сохранённые данные
for (let i = 0; i <= 10; i++) {
    const data = localStorage.getItem(`survey_page_${i}`);
    if (data) {
        console.log(`Page ${i}:`, JSON.parse(data));
    }
}
```

### Очистить все сохранённые данные

```javascript
// Очистить все данные опросника
for (let i = 0; i <= 10; i++) {
    localStorage.removeItem(`survey_page_${i}`);
}
```

### Включить отладку

Добавьте в footer.php:

```javascript
// Отладка
console.log('Current page:', CURRENT_PAGE);
console.log('Restored data:', restoreFromLocalStorage(CURRENT_PAGE));
```
